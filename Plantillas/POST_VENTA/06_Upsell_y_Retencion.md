# MÓDULO A — DETECCIÓN Y PITCH DE UPSELL

## Señales de oportunidad de upsell

*Marcar las que aplican para este cliente:*

### Señales fuertes (actuar en la misma semana):
- ☐ Cliente dijo explícitamente "me gustaría también [X]"
- ☐ Cliente preguntó si podemos hacer algo que está fuera del alcance actual
- ☐ El sistema generó ROI visible y el cliente lo reconoció
- ☐ El negocio del cliente creció y el sistema actual se está quedando pequeño

### Señales medias (plantear en el siguiente checkpoint):
- ☐ Hay un proceso manual visible que se podría automatizar
- ☐ El cliente usa solo el 60% de las funciones del sistema
- ☐ Se abrió una nueva sucursal o canal de ventas
- ☐ El equipo del cliente se queja de carga manual en otra área

### Señales débiles (semilla para mes 3+):
- ☐ El cliente mencionó un problema no relacionado con el sistema actual
- ☐ Hay una temporada alta o evento especial próximo
- ☐ Cambio en el modelo de negocio del cliente

# MÓDULO B — PROTOCOLO DE RETENCIÓN (ANTI-CHURN)

## Niveles de alerta y respuesta

### 🟡 ALERTA AMARILLA — Adopción baja o indiferencia

**Señales:**
- Bot con < 30% del volumen esperado
- Cliente no abre los reportes
- Respuestas lentas o monosilábicas a comunicaciones

**Protocolo:**
```
PASO 1 — Mensaje de check-in genuino (no de venta):
"[NOMBRE], ¿cómo está yendo todo? ¿El sistema te está siendo útil
o hay algo que no está funcionando como esperabas?"

PASO 2 — Si responde con una queja o señal de insatisfacción:
→ Resolver la fricción específica INMEDIATAMENTE, gratis
→ Demostrar con acción, no con palabras

PASO 3 — Reencuadre de valor:
"[NOMBRE], mira esto: en el último mes, aunque el volumen fue menor al estimado,
el sistema igual te ahorró [X horas] y previno [X no-shows]. ¿Cómo podemos
asegurarnos de aprovecharlo más el próximo mes?"

PASO 4 — Ofrecer sesión de refuerzo de 15 minutos:
"¿Qué te parece si nos ponemos 15 minutos para revisar juntos el sistema
y asegurarnos de que lo estás aprovechando al máximo?"
```

---

### 🔴 ALERTA ROJA — Señales claras de cancelación

**Señales:**
- Retainer vencido sin pago
- Cliente dice directamente que quiere cancelar
- Silencio de > 15 días sin responder comunicaciones
- Queja grave sin resolver

**Protocolo de retención de emergencia:**

```
PASO 1 — Contacto de urgencia (dentro de 24 horas):
Canal: Llamada telefónica directa (no WhatsApp)

"[NOMBRE], soy Pablo. Te llamo porque noto que algo no está bien
y quiero entender qué pasó. ¿Tienes 5 minutos?"

PASO 2 — Escuchar sin defenderse:
Preguntar: "¿Qué salió mal desde tu perspectiva?"
NO interrumpir. NO dar excusas. Tomar nota.

PASO 3 — Reconocer y responsabilizarse (si hubo error de Ryventis):
"Tienes razón. [SITUACIÓN] no debió haber pasado así. Eso es responsabilidad nuestra."

PASO 4 — Propuesta de retención:
Opción A (problema técnico): "Déjame resolverlo ahora mismo, gratis, sin compromiso."
Opción B (insatisfacción de valor): "¿Qué tendría que cambiar para que valga la pena seguir?"
Opción C (precio): "¿Si ajustamos el precio a $[X menor] por [X meses], tendría sentido?"

PASO 5 — Si el cliente decide irse de todas formas:
"Lo entiendo perfectamente. Gracias por darnos la oportunidad.
¿Me permites preguntarte qué fue lo que más te faltó? Tu feedback
nos ayuda a mejorar para futuros clientes."
[Terminar bien — este cliente puede referirte aunque no siga contigo]
```
